LinkedHashTable源码解析(JDK1.8.0_131)

  • Java
0 views
2018年 01月11日

1. 数据结构

LinkedHashMap底层实现是HashMap，和HashMap比较起来，增加了一个类似全局的链表，之前HashMap中是有链表的，在table中存储着很多链表，在LinkedHashMap中通过每个节点记录前后节点的方法，实现了把单个链表链接在一起成为一个整的链表，这样我们就可以按照访问顺序或者按照插入顺序来访问节点数据。

2. 继承关系

先看LinkedHashMap的关系图

LinkedHashMap实现了Map接口，继承了HashMap，所以说，LinkedHashMap有很多内容和HashMap是一样的

然后看一下，子类Entry的实现关系

Entry作为基本的节点，可以看到LinkedHashMap的Entry继承自HashMap的Node，在其基础上加上了before和after两个指针，而TreeNode作为HashMap和LinkedHashMap的树节点，继承自LinkedHahsMap的Entry，并且加上了树节点的相关指针，另外提一点：before和parent的两个概念是不一样的，before是相对于链表来的，parent是相对于树操作来的，所以要分两个。
最后看一下迭代器的继承关系

LinkedHashMap的迭代器为遍历节点提供了自己的实现——LinkedHashIterator，对于Key、Value、Entry的3个迭代器，都继承自它。而且内部采用的遍历方式就是在前面提到的Entry里加的新的指向下一个节点的指针after。

3. 构造函数

3.1. 成员变量

看构造函数之前，先看一下LinkedHashMap定义的几个属性


    /**
     * 双链表的头
     */
    transient LinkedHashMap.Entry<K,V> head;

    /**
     * 双链表的尾部
     */
    transient LinkedHashMap.Entry<K,V> tail;

    /**
     * 此链接哈希映射的迭代排序方法：对于访问顺序为true，对于插入顺序为false。
     *
     * @serial
     */
    final boolean accessOrder;

3.2. 构造函数

现在来看构造函数，一共有5个，比HashMap多了一个，因为LinkedHashMap有一个迭代顺序，就是上面提的 accessOrder，LinkedHashMap的构造方法都是用了HashMap的构造器，如果有 initialCapacity 和 loadFactor两个参数，就是用这两个参数初始化HashMap中的Table对象，否则使用默认值，和HashMap完全一致。


    public LinkedHashMap(int initialCapacity, float loadFactor) {
        super(initialCapacity, loadFactor);
        //默认情况下accessOrder为false，也就是按照插入的顺序进行迭代遍历
        accessOrder = false;
    }

    public LinkedHashMap(int initialCapacity) {
        super(initialCapacity);
        accessOrder = false;
    }

    public LinkedHashMap() {
        super();
        accessOrder = false;
    }

    public LinkedHashMap(int initialCapacity,
                         float loadFactor,
                         boolean accessOrder) {
        super(initialCapacity, loadFactor);
        //这里自定义按照插入还是访问顺序进行迭代
        this.accessOrder = accessOrder;
    }

    //使用集合进行初始化，下面我们会看具体的put方法
    public LinkedHashMap(Map<? extends K, ? extends V> m) {
        super();
        accessOrder = false;
        putMapEntries(m, false);
    }

4. 新增元素

我们从上面的 putMapEntries(m, false) 方法继续往下看, 这里LinkedHashMap并没有实现这个方法，直接使用的HashMap的默认实现

final void putMapEntries(Map<? extends K, ? extends V> m, boolean evict) {
        int s = m.size();
        if (s > 0) {
            if (table == null) { // pre-size
                float ft = ((float)s / loadFactor) + 1.0F;
                int t = ((ft < (float)MAXIMUM_CAPACITY) ?
                         (int)ft : MAXIMUM_CAPACITY);
                if (t > threshold)
                    threshold = tableSizeFor(t);
            }
            else if (s > threshold)
                resize();
            for (Map.Entry<? extends K, ? extends V> e : m.entrySet()) {
                K key = e.getKey();
                V value = e.getValue();
                putVal(hash(key), key, value, false, evict);
            }
        }
    }

然后我们继续看 putVal(hash(key), key, value, false, evict)， 这里把代码贴出来，但是不做具体解释了，在前面已经有过详细的描述，这里只分析下面标注的三个有LinkedHashMap具体实现的方法，也就是用这几个方法，LinkedHashMap实现了双链表的存储。

final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
                   boolean evict) {
        Node<K,V>[] tab; Node<K,V> p; int n, i;
        if ((tab = table) == null || (n = tab.length) == 0)
            n = (tab = resize()).length;
        if ((p = tab[i = (n - 1) & hash]) == null)
            //这里需要注意，我们下面会分析，LinkedHashMap有具体实现
            tab[i] = newNode(hash, key, value, null);
        else {
            Node<K,V> e; K k;
            if (p.hash == hash &&
                ((k = p.key) == key || (key != null && key.equals(k))))
                e = p;
            else if (p instanceof TreeNode)
                e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
            else {
                for (int binCount = 0; ; ++binCount) {
                    if ((e = p.next) == null) {
                        p.next = newNode(hash, key, value, null);
                        if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
                            treeifyBin(tab, hash);
                        break;
                    }
                    if (e.hash == hash &&
                        ((k = e.key) == key || (key != null && key.equals(k))))
                        break;
                    p = e;
                }
            }
            if (e != null) { // existing mapping for key
                V oldValue = e.value;
                if (!onlyIfAbsent || oldValue == null)
                    e.value = value;
                //这里需要注意，下面详细分析
                afterNodeAccess(e);
                return oldValue;
            }
        }
        ++modCount;
        if (++size > threshold)
            resize();
        //这里需要注意，下面详细分析
        afterNodeInsertion(evict);
        return null;
    }

看一下 newNode(hash, key, value, null) 在 LinkedHashMap中的实现

    Node<K,V> newNode(int hash, K key, V value, Node<K,V> e) {
        //注意，这里LinkedHashMap.Entry中比HashMap多了 `Entry<K,V> before, after`，这两个节点记录了链表的前后节点，注意，这个和之前的Entry中的next是不同的
        LinkedHashMap.Entry<K,V> p =
            new LinkedHashMap.Entry<K,V>(hash, key, value, e);
        linkNodeLast(p);
        return p;
    }


    // 链接在列表的末尾。
    private void linkNodeLast(LinkedHashMap.Entry<K,V> p) {
        //把最后一个节点放在last中
        LinkedHashMap.Entry<K,V> last = tail;
        //新节点赋值给 tail（最后一个节点）
        tail = p;
        if (last == null)
            //如果原来最后一个为空，那么说明之前没有元素，那么新元素也同时是头
            head = p;
        else {
            //原来的最后一个节点不为空，那么新节点的上一个节点就是 之前的最后一个，两者建立关系
            p.before = last;
            last.after = p;
        }
    }

通过上面的方法插入元素以后，可以保证一点，也是很重要的一点 新元素插入在了链表的结尾，请注意，由于这里是双链表，所以刚才说的链表是迭代器使用的链表，而不是Table中节点中存储的链表。

我们继续看上面说的三个方法的第二个 afterNodeAccess(e) ,这个方法在HashMap中是空方法，在LinkedHashMap有具体的实现。

    void afterNodeAccess(Node<K,V> e) { // 节点移动到最后一个
        LinkedHashMap.Entry<K,V> last;
        //如果是 accessOrder（访问顺序），并且最后一个元素不是当前要操作的元素 
        if (accessOrder && (last = tail) != e) {
            //当前元素赋值给 p, b和a分别为当前元素的前后节点
            LinkedHashMap.Entry<K,V> p = (LinkedHashMap.Entry<K,V>)e, b = p.before, a = p.after;
            //把p的后一个节点置空
            p.after = null;
            //请注意，下面要分析三种情况，前面说了，p b a 原来的顺序是 b --> p --> a ，现在要变成 b --> a --> p

            if (b == null)
                head = a;
            else
                b.after = a;
            if (a != null)
                a.before = b;
            else
                last = b;
            if (last == null)
                head = p;
            else {
                p.before = last;
                last.after = p;
            }
            tail = p;
            ++modCount;
        }
    }

执行完成 afterNodeAccess 方法以后，如果是 按照访问顺序来进行迭代，那么这个被重复放入数据的节点就会被放到链表的结尾。

然后我们看三个方法的最后一个 afterNodeInsertion(evict)

    void afterNodeInsertion(boolean evict) { // possibly remove eldest
        LinkedHashMap.Entry<K,V> first;
        //删除最初数据的条件有多个，前面两个很清楚，一个是传递的参数，一个是head不为空 ，后面一个稍后说明
        if (evict && (first = head) != null && removeEldestEntry(first)) {
            K key = first.key;
            //
            removeNode(hash(key), key, null, false, true);
        }
    }

removeNode 方法是HashMap中的方法，这里不具体分析，只分析 在removeNode的时候调用了一个afterNodeRemoval方法，这个方法在LinkedHashMap有具体实现，主要作用就是解除删除节点的前后关联，重新连接链表的before和after，代码如下

    void afterNodeRemoval(Node<K,V> e) { // unlink
        LinkedHashMap.Entry<K,V> p =
            (LinkedHashMap.Entry<K,V>)e, b = p.before, a = p.after;
        p.before = p.after = null;
        if (b == null)
            head = a;
        else
            b.after = a;
        if (a == null)
            tail = b;
        else
            a.before = b;
    }

然后看一下刚才说 afterNodeInsertion 中有三个条件中的最后 一个 removeEldestEntry

    /**
    * 这里默认返回false，javadoc中说了，这里需要用户重写来满足用户自己的要求，比如下面这样，设置最大为100，当超过100的时候就会把head位置的元素删除
    * private static final int MAX_ENTRIES = 100;
    * protected boolean removeEldestEntry(Map.Entry eldest) {
    *     return size() > MAX_ENTRIES;
    * }
    */
    protected boolean removeEldestEntry(Map.Entry<K,V> eldest) {
        return false;
    }

6. 取值 get

6.1. get(Object key)

/**
     * 返回指定键映射到的值;如果此映射不包含键映射，则返回null。
     *
     * 更正式地说，如果这个映射包含从键k到值v的映射（key == null？k == null：key.equals（k）），那么这个方法返回v; 否则返回null。 （最多可以有一个这样的映射。）
     *
     * null的返回值不一定表示映射不包含该键的映射; 地图也可能明确地将密钥映射到空值。 containsKey操作可以用来区分这两种情况。
     */
    public V get(Object key) {
        Node<K,V> e;
        //调用HashMap的getNode方法来获取节点值
        if ((e = getNode(hash(key), key)) == null)
            return null;
        //如果是按照访问顺序迭代，那么这里会执行上面已经分析的方法 afterNodeAccess，把访问的元素放到最后。
        if (accessOrder)
            afterNodeAccess(e);
        return e.value;
    }

6.2. getOrDefault(Object key, V defaultValue)

除了上面的get之外，LinkedHashMap提供了一个getOrDefault的方法，简单说就是如果没有的话，返回默认值。

    public V getOrDefault(Object key, V defaultValue) {
       Node<K,V> e;
       if ((e = getNode(hash(key), key)) == null)
           return defaultValue;
       if (accessOrder)
           afterNodeAccess(e);
       return e.value;
   }

8. 迭代

我们看一下 LinkedHashIterator，其他两个 key和value 的迭代器都由 LinkedHashIterator来实现

    abstract class LinkedHashIterator {
        //记录下一个节点
        LinkedHashMap.Entry<K,V> next;
        //记录当前节点
        LinkedHashMap.Entry<K,V> current;
        //迭代过程中的修改
        int expectedModCount;

        LinkedHashIterator() {
            //head节点给next
            next = head;
            //修改次数
            expectedModCount = modCount;
            //当前为空
            current = null;
        }

        public final boolean hasNext() {
            //有下一个节点（这里是判断，实际调用nextNode才会返回值）
            return next != null;
        }

        final LinkedHashMap.Entry<K,V> nextNode() {
            //获取next的节点
            LinkedHashMap.Entry<K,V> e = next;
            //如果获取下一个节点的时候修改次数和预期的修改次数不一致了，抛出异常！！！
            if (modCount != expectedModCount)
                throw new ConcurrentModificationException();
            //如果next为null了，抛出异常
            if (e == null)
                throw new NoSuchElementException();
            //当前节点复制
            current = e;
            next = e.after;
            return e;
        }

        public final void remove() {
            //p定义为当前节点
            Node<K,V> p = current;
            //p为空，抛出异常，不能删除
            if (p == null)
                throw new IllegalStateException();
            //修改次数和当前不一致，抛出异常
            if (modCount != expectedModCount)
                throw new ConcurrentModificationException();
            //赋值，删除实际的节点
            current = null;
            K key = p.key;
            removeNode(hash(key), key, null, false, false);
            expectedModCount = modCount;
            //重新赋值修改次数
        }
    }
