ArrayList源码解析(JDK1.8.0_131)

  • Java
0 views
2018年 01月12日

1. 数据结构

ArrayList 的实现是依靠数组来完成的。
List接口的可调整大小的实现。实现List的所有可选操作，并允许所有元素，包括null。除了实现List接口之外，该类还提供了一些方法来处理内部用来存储列表的数组的大小。 （这个类大致相当于Vector，除了它是不同步的。）
Size, isEmpty, get, set, iterator, listIterator运行时间是一定的。 add操作以分摊的恒定时间运行，即添加n个元素需要O（n）个时间。所有其他的操作都在线性时间内运行（粗略地说）。与LinkedList实现相比，常数因子较低。

2. 构造函数

成员变量


    /**
     * 默认的初始化容量
     */
    private static final int DEFAULT_CAPACITY = 10;

    /**
     * 用于空实例的共享空数组实例。（比如，初始化的容量为0时，初始化对象就会直接被赋值成 EMPTY_ELEMENTDATA）
     */
    private static final Object[] EMPTY_ELEMENTDATA = {};

    /**
     * 共享空数组实例，用于默认大小的空实例。我们将其与空的ELEMENTDATA区分开来，以便在添加第一个元素时知道要膨胀多少。
     */
    private static final Object[] DEFAULTCAPACITY_EMPTY_ELEMENTDATA = {};

    /**
     * 存储数组列表元素的数组缓冲区。
     * 数组列表的容量是这个数组缓冲区的长度。
     * 空的ArrayList（elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA），当添加第一个元素时，将会被扩展成 DEFAULT_CAPACITY。
     */
    transient Object[] elementData; // non-private to simplify nested class access

    /**
     * 数组列表的大小(它包含的元素的数量)。
     *
     * @serial
     */
    private int size;

构造函数

ArrayList()

    /**
     * 使用默认的空对象初始化
     */
    public ArrayList() {
        this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA;
    }

ArrayList(int initialCapacity)

    /**
     * 使用指定的大小初始化对象
     *
     * @param  initialCapacity  list的初始化大小
     * @throws IllegalArgumentException 如果指定的初始容量为负。
     */
    public ArrayList(int initialCapacity) {
        if (initialCapacity > 0) {
            //如果初始化大小大于0，那么生成一个Object数组，大小为 initialCapacity
            this.elementData = new Object[initialCapacity];
        } else if (initialCapacity == 0) {
            //如果初始化大小为0，使用 EMPTY_ELEMENTDATA 初始化
            this.elementData = EMPTY_ELEMENTDATA;
        } else {
            //如果为负数，抛出异常
            throw new IllegalArgumentException("Illegal Capacity: "+
                                               initialCapacity);
        }
    }

ArrayList(Collection<? extends E> c)

    /**
     * 构造一个包含初始化元素的list。
     * 元素的顺序是由初始化元素的迭代顺序来决定的。
     * @param c 将被放置到这个列表中的集合。
     * @throws NullPointerException 集合为空时抛出异常
     */
    public ArrayList(Collection<? extends E> c) {
        //传入参数，转成新的数组对象，赋值给 elementData
        elementData = c.toArray();
        if ((size = elementData.length) != 0) {
            // c.toArray might (incorrectly) not return Object[] (see 6260652)
            // 关于上面这个6260652bug，大体的意思是，如果集合元素是Object的子类，如String,toArray返回的数组也是String,而不是Object,所以这里需要进一步的转成Object类型
            if (elementData.getClass() != Object[].class)
                elementData = Arrays.copyOf(elementData, size, Object[].class);
        } else {
            // 数组长度为0，使用默认的空对象代替
            this.elementData = EMPTY_ELEMENTDATA;
        }
    }

这里需要看一下toArray方法，这是所有实现Collection接口的集合中都存在的方法，ArrayList实现了List接口，List继承了Collection接口，所以不例外的ArrayList中也实现了toArray方法，我们这里看一下ArrayList具体的实现方法。简单的说就是返回了一个新的数组对象。

/**
* 以适当的顺序（从第一个元素到最后一个元素）返回包含此列表中所有元素的数组。
* 返回的数组将是“安全的”，因为这个列表没有引用它。 （换句话说，这个方法必须分配一个新的数组）。 调用者可以自由修改返回的数组。
* 此方法充当基于数组和基于集合的API之间的桥梁。 
*/
public Object[] toArray() {
        return Arrays.copyOf(elementData, size);
    }

3. 新增

add(E e)

add方法是最常用的方法，简单说就是每次把新元素放在数组的结尾。
添加之前先判断数组容量，如果容量不足就扩展，扩展时会比较当前容量的1.5倍和当前对象数量+1,然后根据容量生成一下新的数组并且返回，返回后真正执行添加在末尾的操作。


    /**
     * 将指定的元素附加到此列表的末尾。
     *
     * @param e 将要添加的元素
     * @return <tt>true</tt> (as specified by {@link Collection#add})
     */
    public boolean add(E e) {
        //确保数组可以容纳新的元素（size是当前的数组大小，size + 1表示添加新元素需要的容量）
        ensureCapacityInternal(size + 1);  // Increments modCount!!
        //把新元素放在数组的末尾
        elementData[size++] = e;
        return true;
    }

    private void ensureCapacityInternal(int minCapacity) {
        //如果当前是默认的空对象，重新定义最小值，最小值不应该小于默认值10
        if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
            minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
        }
        //扩展容量
        ensureExplicitCapacity(minCapacity);
    }

    private void ensureExplicitCapacity(int minCapacity) {
        modCount++;

        // overflow-conscious code
        //如果目标最小容量大于当前数组长度，进行扩容
        if (minCapacity - elementData.length > 0)
            grow(minCapacity);
    }

    private void grow(int minCapacity) {
        // overflow-conscious code
        int oldCapacity = elementData.length;
        // 新的容量是当前的 1.5 倍
        int newCapacity = oldCapacity + (oldCapacity >> 1);
        // 如果新容量小于目标最小容量，那么直接把最小目标容量赋值给新容量
        if (newCapacity - minCapacity < 0)
            newCapacity = minCapacity;
        如果超过最大容量 2^31 - 8
        if (newCapacity - MAX_ARRAY_SIZE > 0)
            newCapacity = hugeCapacity(minCapacity);
        // minCapacity is usually close to size, so this is a win:
        // 请注意，执行Arrays.copyOf后，对象成了一个新的对象，和原来的对象没有引用关系
        elementData = Arrays.copyOf(elementData, newCapacity);
    }

    private static int hugeCapacity(int minCapacity) {
        //如果目标最小容量为负值，说明溢出了，超过了2^31-1以后为负值，抛出异常
        if (minCapacity < 0) // overflow
            throw new OutOfMemoryError();
        //如果只是大于默认的最大值，赋值为Int的最大值（注意，这里JavaDoc中描述，有些虚拟机在超过 2^31 - 8 会出现OutOfMemoryError）
        return (minCapacity > MAX_ARRAY_SIZE) ?
            Integer.MAX_VALUE :
            MAX_ARRAY_SIZE;
    }

add(int index, E element)

    /**
     * 将指定的元素插入此列表中的指定位置。 将当前位置的元素（如果有的话）和任何随后的元素移到右侧（在其索引中增加一个）。
     *
     * @param index 指定元素将要被插入的位置
     * @param element 将要插入的元素
     * @throws IndexOutOfBoundsException {@inheritDoc}
     */
    public void add(int index, E element) {
        //保证index是有效值
        rangeCheckForAdd(index);
        //确保数组的容量足够容纳 size + 1 个元素，否则进行扩展
        ensureCapacityInternal(size + 1);  // Increments modCount!!
        // 复制，下面依次是参数的含义
        // 源数组
        // 需要复制的源数组数据的index开始值
        // 目标数组
        // 复制数据到目标数组的index开始值
        // 需要复制大目标数据的元素数量
        System.arraycopy(elementData, index, elementData, index + 1,
                         size - index);
        复制以后，index位置直接赋值
        elementData[index] = element;
        size++;
    }

    /**
     * 用于添加和添加所有内容的范围检查的版本。
     */
    private void rangeCheckForAdd(int index) {
        //保证插入的index值是有效的
        if (index > size || index < 0)
            throw new IndexOutOfBoundsException(outOfBoundsMsg(index));
    }

addAll(Collection<? extends E> c)

    public boolean addAll(Collection<? extends E> c) {
        //源集合转换成数组
        Object[] a = c.toArray();
        int numNew = a.length;
        //确保目标容量，如果不足，进行扩容
        ensureCapacityInternal(size + numNew);  // Increments modCount
        //同上，复制数组
        System.arraycopy(a, 0, elementData, size, numNew);
        //元素数量更新
        size += numNew;
        return numNew != 0;
    }

addAll(int index, Collection<? extends E> c)

    public boolean addAll(int index, Collection<? extends E> c) {
        //检测插入的index目标值有效
        rangeCheckForAdd(index);
        
        Object[] a = c.toArray();
        int numNew = a.length;
        ensureCapacityInternal(size + numNew);  // Increments modCount
        //计算需要移动的数组元素数量，如果需要，进行元素移动
        int numMoved = size - index;
        if (numMoved > 0)
            System.arraycopy(elementData, index, elementData, index + numNew,
                             numMoved);
        //把新插入元素复制到目标位置
        System.arraycopy(a, 0, elementData, index, numNew);
        size += numNew;
        return numNew != 0;
    }

获取

ArrayList只有一个获取元素的方法， 方法也很简单，校验有效性，然后直接返回数据

    public E get(int index) {
        //校验index值有效
        rangeCheck(index);
        //直接返回数组元素
        return elementData(index);
    }

删除

remove(int index)

删除方法很简单，校验index有效性，把后续元素复制到前一个位置，然后，数组数量-1，数组的最后一个位置置空。最后返回旧值。

/**
     * 在这个列表中删除指定位置的元素。将任何后续元素移到左边(从它们的索引中减去一个)。
     *
     * @param index the index of the element to be removed
     * @return the element that was removed from the list
     * @throws IndexOutOfBoundsException {@inheritDoc}
     */
    public E remove(int index) {
        rangeCheck(index);

        modCount++;
        E oldValue = elementData(index);

        int numMoved = size - index - 1;
        if (numMoved > 0)
            System.arraycopy(elementData, index+1, elementData, index,
                             numMoved);
        elementData[--size] = null; // clear to let GC do its work

        return oldValue;
    }

remove(Object o)

如果指定的元素存在，那么删除这个元素在List中第一次出现的位置元素，如果不存在，不进行删除操作。

    public boolean remove(Object o) {
        if (o == null) {
            for (int index = 0; index < size; index++)
                if (elementData[index] == null) {
                    fastRemove(index);
                    return true;
                }
        } else {
            for (int index = 0; index < size; index++)
                if (o.equals(elementData[index])) {
                    fastRemove(index);
                    return true;
                }
        }
        return false;
    }

    /*
     * 私有删除方法，跳过边界检查，不返回删除的值。（这里基本类似上上面的 remove(index)方法 ）
     */
    private void fastRemove(int index) {
        modCount++;
        int numMoved = size - index - 1;
        if (numMoved > 0)
            System.arraycopy(elementData, index+1, elementData, index,
                             numMoved);
        elementData[--size] = null; // clear to let GC do its work
    }

复制

从JavaDoc可以看出，这是一个浅拷贝。实现也很简单，复制一个新的对象，给每个元素赋值。

    /**
     * 返回这个< tt >数组列表</ tt >实例的一个浅拷贝。(这些元素本身并没有被复制。)
     *
     * @return a clone of this <tt>ArrayList</tt> instance
     */
    public Object clone() {
        try {
            ArrayList<?> v = (ArrayList<?>) super.clone();
            v.elementData = Arrays.copyOf(elementData, size);
            v.modCount = 0;
            return v;
        } catch (CloneNotSupportedException e) {
            // this shouldn't happen, since we are Cloneable
            throw new InternalError(e);
        }
    }

迭代

forEach(Consumer<? super E> action)

这个方法比较特殊，通常我们遍历使用 fori, foreach等，这里有一个特殊的 forEach， 比如我们有一个对象，实现了 java.util.function.Consumer ，然后在对ArrayList进行遍历的时候，会对数组中的对象主动调用 accept 方法，直到结束或者抛出异常。

    public void forEach(Consumer<? super E> action) {
        Objects.requireNonNull(action);
        final int expectedModCount = modCount;
        @SuppressWarnings("unchecked")
        final E[] elementData = (E[]) this.elementData;
        final int size = this.size;
        for (int i=0; modCount == expectedModCount && i < size; i++) {
            action.accept(elementData[i]);
        }
        if (modCount != expectedModCount) {
            throw new ConcurrentModificationException();
        }
    }

这里有一个例子：

import java.util.*;
import java.util.function.*;

class MyConsumer<T> implements Consumer<T>{
 public void accept(T ctask){
  System.out.println("Processing Task " + ctask);
 }
}

import java.util.*;

public class test {
   public static void main(String[] args) 
   {
  ArrayList<Integer> myList;
  MyConsumer<Integer> mcons;
  
  myList = new ArrayList<> ();
  mcons = new MyConsumer<Integer>();
  
  myList.add(100);
  myList.add(200);
  myList.add(300);
  
  myList.forEach(mcons);
 }
}

Processing Task 200
Processing Task 300

4. 其他

线程安全

ArrayList不是线程安全的，如果需要，请使用下面的方式获取安全实例

List list = Collections.synchronizedList(new ArrayList(...));

fail-fast

这个类的迭代器和listIterator方法返回的迭代器是快速失败的：如果在迭代器被创建之后，在任何时候结构上修改了列表，除了通过迭代器自己的remove或add方法以外，迭代器将抛出一个ConcurrentModificationException异常。

因此，在并发修改的情况下，迭代器快速而干净地失败，而不是在未来的未定的时间冒着任意的，不确定的行为冒险。

请注意，迭代器的故障快速行为无法得到保证，因为一般来说，不可能在存在非同步并发修改的情况下做出任何硬性保证。 失败快速迭代器尽最大努力抛出ConcurrentModificationException。 因此，编写一个依赖于这个异常的程序是正确的：迭代器的快速失败行为只能用来检测错误。
