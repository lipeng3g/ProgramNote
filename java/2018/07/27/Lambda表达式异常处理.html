1. 概览

在Java 8中，Lambda Expressions通过提供表达行为的简洁方式开始促进函数式编程。但是，JDK提供的功能接口不能很好地处理异常 - 在处理异常时代码变得冗长和繁琐。

在本文中，我们将探讨在编写lambda表达式时处理异常的一些方法。

2. 处理未经检查的异常

首先，我们用一个例子来理解这个问题。 当前有个List<Integer>，我们用一个常量，比如50来除以每个元素，并打印结果：

List<Integer> integers = Arrays.asList(3, 9, 7, 6, 10, 20);
integers.forEach(i -> System.out.println(50 / i));

上面的程序是有效的，但是，请注意，这里有一个问题，当某一个元素为 0 的时候，可能会抛出ArithmeticException异常。
下面让我们使用try-catch来解决这个问题：

List<Integer> integers = Arrays.asList(3, 9, 7, 0, 10, 20);
integers.forEach(i -> {
    try {
        System.out.println(50 / i);
    } catch (ArithmeticException e) {
        System.err.println(
          "Arithmetic Exception occured : " + e.getMessage());
    }
});

try-catch解决了异常的问题，但是Lambda表达式的简洁性就不明显了，它不再像原来的那个简洁的小函数了。
为了解决这个问题，我们需要为Lambda函数写一个Lambda包装器，代码如下：

static Consumer<Integer> lambdaWrapper(Consumer<Integer> consumer) {
    return i -> {
        try {
            consumer.accept(i);
        } catch (ArithmeticException e) {
            System.err.println(
              "Arithmetic Exception occured : " + e.getMessage());
        }
    };
}

使用方法如下：

List<Integer> integers = Arrays.asList(3, 9, 7, 0, 10, 20);
integers.forEach(lambdaWrapper(i -> System.out.println(50 / i)));

首先，我们编写了一个包装器方法，负责异常处理，然后将Lambda表达式作为参数传递给此方法。
包装器方法按照预期的结果工作，但是你可能为认为，它难道不是把Lambda表达式中的try-catch块移动到了一个单独的方法中了吗？它实际并没有减少代码量。

当我们针对某个特殊案例编写上面的代码的时候，确实存在上面我们所提到的问题，实际并没有减少代码量。但是，我们可以使用泛型来改进此方法，并把它应用于其他场景。

static <T, E extends Exception> Consumer<T>
  consumerWrapper(Consumer<T> consumer, Class<E> clazz) {
  
    return i -> {
        try {
            consumer.accept(i);
        } catch (Exception ex) {
            try {
                E exCast = clazz.cast(ex);
                System.err.println(
                  "Exception occured : " + exCast.getMessage());
            } catch (ClassCastException ccEx) {
                throw ex;
            }
        }
    };
}

然后我们可以这样使用：

List<Integer> integers = Arrays.asList(3, 9, 7, 0, 10, 20);
integers.forEach(
  consumerWrapper(
    i -> System.out.println(50 / i), 
    ArithmeticException.class));

正如我们所看到的，我们的包装器方法的这个迭代需要两个参数，lambda表达式和要捕获的Exception类型。这个lambda包装器能够处理所有数据类型，而不仅仅是Integers，并捕获任何特定类型的异常，而不是超类Exception。

另请注意，我们已将方法的名称从lambdaWrapper更改为consumerWrapper。这是因为此方法仅处理Consumer类型的Functional Interface的lambda表达式。我们可以为其他功能接口编写类似的包装方法，如Function，BiFunction，BiConsumer等。

3. 处理检查异常

Java中的异常分为受检查的异常和不受检查的异常。

  • 受检查的异常：这种在编译时被强制检查的异常称为”受检查的异常”。即在方法的声明中声明的异常。
  • 不受检查的异常：在方法的声明中没有声明，但在方法的运行过程中发生的各种异常被称为”不被检查的异常”。这种异常是错误，会被自动捕获。

我们继续上一环节的示例，但是这一次我们不再把数据打印在控制台，而是写入文件。写入操作可能会抛出IOException.

static void writeToFile(Integer integer) throws IOException {
    // logic to write to file which throws IOException
}

List<Integer> integers = Arrays.asList(3, 9, 7, 0, 10, 20);
integers.forEach(i -> writeToFile(i));

程序编译时，可能会得到下面的异常： java.lang.Error: Unresolved compilation problem: Unhandled exception type IOException . 由于IOException属于受检查的异常，它必须被处理。现在我们有两种方式处理：

  • =抛出异常并在其他地方处理它
  • 在具有lambda表达式的方法内处理它

3.1. 从Lambda表达式抛出检查异常

让我们从写入lambda表达式的方法中抛出异常，在本例中为main:

public static void main(String[] args) throws IOException {
    List<Integer> integers = Arrays.asList(3, 9, 7, 0, 10, 20);
    integers.forEach(i -> writeToFile(i));
}

但是，编译时，我们会得到未处理的IOException.这是因为Lambda表达式类似于匿名的内部类。在这种情况下，Lambda表达式是一个实现了Consumer<T>接口的accept(T t)的对象。

Consumer<Integer> consumer = new Consumer<Integer>() {
  
    @Override
    public void accept(Integer integer) throws Exception {
        writeToFile(integer);
    }
};

上面的代码不能编译通过，因为accept方法的实现不能抛出任何异常。 最直接的方法是使用try-catch并将已检查的异常包装到未经检查的异常中并重新抛出：

List<Integer> integers = Arrays.asList(3, 9, 7, 0, 10, 20);
integers.forEach(i -> {
    try {
        writeToFile(i);
    } catch (IOException e) {
        throw new RuntimeException(e);
    }
});

上面的方法可以编译并且运行，但与上一节中未经检查的异常情况下的示例具有相同的问题。 因为我们只想抛出异常，所以我们需要编写自己的消费者功能接口，它可以抛出异常，然后使用它的包装器方法。我们称之为ThrowingConsumer：

@FunctionalInterface
public interface ThrowingConsumer<T, E extends Exception> {
    void accept(T t) throws E;
}

static <T> Consumer<T> throwingConsumerWrapper(
  ThrowingConsumer<T, Exception> throwingConsumer) {
  
    return i -> {
        try {
            throwingConsumer.accept(i);
        } catch (Exception ex) {
            throw new RuntimeException(ex);
        }
    };
}

现在我们可以编写我们的lambda表达式，它可以抛出异常而不会失去简洁性。

List<Integer> integers = Arrays.asList(3, 9, 7, 0, 10, 20);
integers.forEach(throwingConsumerWrapper(i -> writeToFile(i)));

3.2. 在Lambda中处理检查异常

我们可以修改包装器来处理受检查的异常。由于我们的ThrowingConsumer使用泛型，我们可以处理任何异常。

static <T, E extends Exception> Consumer<T> handlingConsumerWrapper(
  ThrowingConsumer<T, E> throwingConsumer, Class<E> exceptionClass) {
  
    return i -> {
        try {
            throwingConsumer.accept(i);
        } catch (Exception ex) {
            try {
                E exCast = exceptionClass.cast(ex);
                System.err.println(
                  "Exception occured : " + exCast.getMessage());
            } catch (ClassCastException ccEx) {
                throw new RuntimeException(ex);
            }
        }
    };
}

可以在我们的示例中使用此包装器来处理IOException并通过将它们包装在未经检查的异常中来抛出任何其他已检查的异常：

List<Integer> integers = Arrays.asList(3, 9, 7, 0, 10, 20);
integers.forEach(handlingConsumerWrapper(
  i -> writeToFile(i), IOException.class));

类似于其他的检查异常，其他的ThowingFunction，ThrowingBiFunction也可以使用这种方式进行处理。

4. 总结

在本文中，我们介绍了如何使用包装器方法处理lambda表达式中的特定异常，而不会失去简洁性。我们还学习了如何为JDK中存在的功能接口编写抛出替代方法，以通过将它们包装在未经检查的异常中或处理它们来抛出已检查的异常。
